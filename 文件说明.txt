ESP32-S3-EYE 项目文件夹说明

本文档说明以下两个目录中的主要文件和子文件夹分别是做什么的：
1. D:\prj\esp-bsp-master\examples\display
2. D:\MyHomework\PavementDamagesG-7\esp32s3_eye_watch

说明原则：
- 两个目录的整体结构非常接近。
- display 是 ESP-BSP 自带的显示示例工程。
- esp32s3_eye_watch 是基于 display 改写出来的手表工程。
- build、managed_components、.vscode 这类目录主要是构建或开发环境自动生成/使用的辅助目录。

==================================================
一、D:\prj\esp-bsp-master\examples\display 文件夹说明
==================================================

1. .vscode/
   VS Code 工程配置目录，通常存放调试、任务、工程设置等文件，方便在 VS Code 中开发和运行项目。

2. build/
   编译输出目录。
   运行 idf.py build、idf.py flash 等命令后生成，里面通常包含：
   - .o 目标文件
   - .elf/.bin 固件文件
   - map 文件
   - ninja/cmake 中间文件
   一般不手工修改。

3. doc/
   文档资源目录。
   这个示例中通常放演示截图、预览图等文件，用于 README 或文档展示。

4. main/
   主应用源码目录，是工程最重要的目录之一。
   里面存放应用入口、LVGL 界面代码、组件配置等。

5. managed_components/
   ESP-IDF 组件管理器自动下载和缓存的依赖组件目录。
   例如通过 idf_component.yml 声明的组件依赖，编译时会解析并放到这里。
   一般不手工改。

6. build_output.txt
   构建输出日志文件。
   用来记录编译过程、报错信息、警告信息或构建结果，方便排查问题。

7. CMakeLists.txt
   工程顶层 CMake 配置文件。
   作用：
   - 指定最低 CMake 版本
   - 引入 ESP-IDF 的 project.cmake
   - 定义项目名（例如 project(display)）
   它决定整个工程如何被 ESP-IDF 构建系统识别。

8. dependencies.lock
   依赖锁定文件。
   用来记录当前项目解析后的组件依赖版本，确保不同时间构建时尽量使用一致的依赖版本。

9. null
   占位或测试产生的空文件。
   对主要业务逻辑通常没有实际作用，可以视为无功能占位文件。

10. partitions.csv
    Flash 分区表文件。
    定义 ESP32 芯片中各个分区的布局，例如：
    - NVS
    - PHY init
    - factory app
    - 其他数据区
    固件烧录时会按这个分区表组织 Flash 空间。

11. pytest_display.py
    Python 自动化测试脚本。
    一般用于配合 pytest 或 ESP-IDF 的自动测试流程，对 display 示例做验证。

12. README.md
    工程说明文档。
    主要介绍：
    - 这个示例是做什么的
    - 如何选择板卡配置
    - 如何用 idf.py 编译和烧录

13. run_display.bat
    Windows 批处理脚本。
    通常用于简化编译、烧录或运行流程，避免每次手敲完整命令。

14. sdkconfig
    当前工程实际使用的配置文件。
    保存 menuconfig 或构建过程中最终生效的 Kconfig 配置，例如：
    - 芯片型号
    - PSRAM 配置
    - FreeRTOS 配置
    - LVGL/显示相关配置

15. sdkconfig.old
    上一次配置备份文件。
    当 sdkconfig 发生变化时，旧版本可能会保存在这里，便于比较配置差异。

16. sdkconfig.bsp.*
    各种开发板对应的默认配置文件。
    例如：
    - sdkconfig.bsp.esp32_s3_eye
    - sdkconfig.bsp.esp-box
    - sdkconfig.bsp.m5stack_core_s3
    作用是让同一个示例工程可以快速切换到不同板卡上编译。
    对 ESP32-S3-EYE 来说，最关键的是：
    - sdkconfig.bsp.esp32_s3_eye

17. sdkconfig.defaults.esp32
    针对某一类芯片或平台的默认配置模板。
    构建时可与其他 sdkconfig 默认文件一起使用。

--------------------------------------------------
display\main 目录说明
--------------------------------------------------

1. images/
   图片资源目录。
   通常存放界面显示需要的图片、Logo 等资源。

2. CMakeLists.txt
   main 组件的构建脚本。
   指定 main 目录下哪些源文件参与编译。

3. idf_component.yml
   main 组件依赖配置文件。
   作用：
   - 声明依赖哪个 BSP 板级支持包
   - 声明依赖哪些组件（例如 esp_lvgl_port）
   - 让 ESP-IDF 组件管理器自动解析依赖

4. lvgl_demo_ui.c
   LVGL 界面实现文件。
   主要负责：
   - 创建屏幕上的对象
   - 设置颜色、位置、文字、图片
   - 组织示例界面布局

5. lvgl_demo_ui.c.original
   原始备份文件。
   一般用来保留最初版本，便于对比修改前后的差异。

6. main.c
   应用入口文件。
   一般包含 app_main()，负责：
   - 初始化显示
   - 启动 BSP
   - 初始化 LVGL
   - 调用 lvgl_demo_ui.c 中的界面创建函数

7. main.c.original
   原始备份文件。
   用于保留修改前的入口程序版本。

--------------------------------------------------
esp32s3_eye_watch\main 目录说明
--------------------------------------------------

1. CMakeLists.txt
   main 组件的编译注册文件。
   当前内容表示：
   - 源文件包括 main.c 和 lvgl_demo_ui.c
   - 依赖的私有组件包括 esp_timer、esp_lcd、esp_lvgl_port、button 等
   也就是说，这个文件决定“手表逻辑代码如何参与编译”。

2. idf_component.yml
   组件依赖声明文件。
   当前工程里它的作用非常关键：
   - 指定依赖 esp32_s3_eye BSP
   - 通过 override_path 指向本地 BSP 路径：
     d:/prj/esp-bsp-master/bsp/esp32_s3_eye
   - 声明依赖 espressif/esp_lvgl_port
   这说明该工程直接复用了本地 ESP-BSP 仓库中的 ESP32-S3-EYE 板级支持包。

3. main.c
   手表工程主入口文件，是整个应用的启动控制中心。
   从代码内容看，它主要负责：
   - 定义两个按键 GPIO：
     * GPIO0：切换手表样式
     * GPIO14：切换手表界面和流体界面
   - 创建两个信号量 button1_sem / button2_sem
   - 注册 GPIO 中断服务
   - 在中断中只发送信号量，不直接操作 LVGL
   - 创建 button_handler_task 任务处理按键事件
   - 在任务中加锁调用：
     * watch_switch_style()
     * watch_switch_ui()
   - 启动显示和背光
   - 调用 example_lvgl_demo_ui(scr) 创建初始界面

   简单理解：
   main.c 是“系统初始化 + 按键事件调度中心”。

4. main.c.original
   main.c 的原始备份文件。
   用来保存改写前的参考版本，便于对比。

5. lvgl_demo_ui.c
   手表界面与流体界面的核心实现文件，是当前项目最核心的界面逻辑文件。
   从代码内容看，它主要负责：
   - 定义 UI 状态：
     * UI_STATE_WATCH：手表界面
     * UI_STATE_FLUID：流体界面
   - 保存当前表盘样式编号 g_watch_style
   - 保存当前时间戳 g_current_ts
   - 创建和管理 LVGL 定时器：
     * g_watch_timer：每秒更新时间
     * g_fluid_timer：驱动流体动画刷新
   - 创建手表界面中的标签：
     * 时间标签
     * 日期标签
     * 星期标签
   - 定义四种主题颜色：
     * Modern（蓝色）
     * Classic（金色）
     * Minimal（黑白）
     * Digital（绿色）
   - 初始化固定起始时间：2026-04-08 12:00:00
   - 每秒更新时间显示
   - 创建手表主界面布局：
     * 标题 ESP32-S3-EYE
     * 中间时间
     * 日期
     * 星期数字 3
     * 底部两个圆点
   - 实现流体模拟数据结构：
     * 500 个粒子的位置和速度数组
     * 画布缓冲区
   - 实现流体初始化、粒子更新、边界碰撞和绘制

   简单理解：
   lvgl_demo_ui.c 是“界面绘制 + 时间更新 + 流体动画”的核心文件。

6. lvgl_demo_ui.c.original
   lvgl_demo_ui.c 的原始备份文件。
   用于保留最初示例界面代码，方便回退和比较。

7. images/
   图片资源目录。
   如果工程中有 Logo、图标、图片转码后的资源文件，通常放在这里。
